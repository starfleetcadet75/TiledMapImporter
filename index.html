TiledMapImporter

A C# library for using custom maps made with the Tiled Map Editor in MonoGame/Xna.

View project on GitHub

Background

This project developed out of my own need to import maps made using Tiled into my MonoGame projects. I originally tried using some of the libraries already out there but none seem to work well enough. Therefore, I created my own.

The Tiled Map Editor is a great piece of free software that you can find here. In order to parse maps saved in JSON format, I use JSON.net which can be found here. This project is written in C# and is designed to be used in MonoGame. While it has been discontinued, Microsoft's XNA 4.0 should also work well with it.

Quick Start

You can download the zip file which includes a DemoGame project that demonstrates the library's capabilities.

If you want to add the library to your project, you can download the MapImporter.dll file from the release page, add a reference to it in your project, then follow these steps:

  1. Use NuGet to download the latest version of Newtonsoft.Json into your project. (Forgetting this step will lead to null reference exceptions!)

  2. Add a using statement:
    using MapImporter

  3. Create a new Map object by calling this line:
    Map map = Importer.ImportMap(@"Content/NewBarkTown.json")

  4. Because the tileset images (which should be in XNB format somewhere inside your content folder) must be loaded as Texture2D objects, you have to call this line directly after creating a new map object:
    map.LoadTilesetTextures(Content)

  5. Finally, draw the Tiled map in your game by calling the following inside your Draw method:
    map.Draw(spriteBatch, graphics.GraphicsDevice.Viewport.Bounds, new Vector(0, 0))

Keep in mind this is the quick start. There are tons of other features that are documented in the Documentation pages.

For use in XNA Game Studio 4.0: Download the latest release of the build specifically for XNA. The MonoGame build depends on some references that MonoGame uses which are not available in the same way inside XNA. The code is exactly the same.
